# Agents and Environments
## Agents and Environments
- **Agent**: Anything that perceives its environment through sensors and acts upon that environment through actuators.
- **Environment**: The setting in which an agent operates.
- **Sensor**: Device that provides perceptual inputs to the agent.
- **Actuator**: Device that enables the agent to perform actions in the environment.

### Agent Function and Percept Sequence
- **Percept**: The agent's current sensory input at any given instant.
- **Percept Sequence**: The complete history of all percepts the agent has ever experienced.
- **Agent Function**: A mapping from any percept sequence to an action. Mathematically, the agent's behavior is described by this function.
- **Agent Program**: The concrete implementation of the agent function within a physical system.

### Example: Vacuum-Cleaner World
- **World Description**: A simple world with two locations, A and B, where the agent can perceive the current location and whether it is dirty.
- **Agent Actions**: The agent can move left, move right, suck the dirt, or do nothing.
- **Agent Function Example**: If the current square is dirty, then suck; otherwise, move to the other square.
- **Partial Tabulation**: A table illustrating the agent function for various percept sequences (Fig. 2.3).
- **Agent Program**: An implementation of the agent function (Fig. 2.8 on page 48).

### Importance of the Agent Function
- **Agent Function vs. Agent Program**: The agent function is an abstract mathematical description, while the agent program is a concrete implementation.
- **Evaluation**: The proper way to fill out the table defines an agent's quality, intelligence, or stupidity, which is discussed in the next section.

### Tools for Analysis
- The concept of an agent is a tool for analyzing systems, not an absolute categorization.
- Examples like a calculator can be viewed as agents, but this doesn’t aid in understanding the system’s functionality.
- AI focuses on designing artifacts that interact with the world, particularly those with significant computational resources and nontrivial decision-making tasks.

# Good Behavior: The Concept of Rationality
## Section 2.2. Good Behavior: The Concept of Rationality
### Definition of a Rational Agent
- A rational agent performs actions that lead to optimal outcomes in its environment.
- The performance of the agent is evaluated through a performance measure that assesses the sequence of environment states, not the agent's internal states.

### Performance Measure
- A performance measure evaluates the sequence of environment states to determine the agent's effectiveness.
- Example: For a vacuum-cleaner agent, using a measure based solely on the amount of dirt cleaned might lead to counterproductive behaviors like repeatedly cleaning and re-distributing dirt.

### Designing Performance Measures
- Performance measures should align with the desired outcomes in the environment.
- Example: A more appropriate measure would reward a clean floor, considering factors like electricity consumption and noise generation.

### Challenges in Performance Measurement
- Average performance can be misleading, as different agents may achieve the same average performance through different means.
- Example: One agent might clean energetically with frequent breaks, while another cleans at a steady pace.
- Philosophical implications: Questions of preference and value, such as whether a reckless life with highs and lows is better than a safe but humdrum existence, remain open to interpretation.

These points capture the essence of the concept of rationality and the challenges in designing appropriate performance measures for agents.

# Intelligent Agents
## Rationality
### Definition and Components
- **Rationality**: The performance measure, the agent's prior knowledge, available actions, and percept sequence.
- **Definition of a Rational Agent**: Selects actions to maximize its performance measure based on percept sequence and built-in knowledge.

## Example: Simple Vacuum Cleaner Agent
- **Agent Function**: Cleans a square if it is dirty; moves to the other square if not.
- **Assumptions**:
  - Performance measure: One point per clean square over 1000 time steps.
  - Known geography, unknown dirt distribution and initial location.
  - Actions: Left, Right, Suck.
  - Perceptions: Correctly perceives location and dirt status.
- **Rationality**: The agent is rational under these conditions, as its expected performance is at least as high as any other agent’s. Exercise 2.2 requires proving this.

- **Irrationality**: The agent would be irrational if:
  - All dirt is cleaned, and the performance measure penalizes movements.
  - Clean squares can become dirty again, requiring periodic re-cleaning.
  - Unknown environment geography, necessitating exploration.

# Rationality, Learning, and Autonomy

## Omnipotence, Learning, and Autonomy

### Distinguishing Rationality from Omniscience

- We must distinguish between rationality and omniscience. An omniscient agent knows the actual outcome of its actions and can act accordingly, but omniscience is impossible in reality.
- Consider the example of walking on the Champs Elysées and encountering an old friend. You decide to cross the street because there is no traffic, acting rationally. However, at 33,000 feet, a cargo door falls off a passing airliner, and before you cross, you are killed.
- Was your decision to cross the street irrational? It is unlikely that your obituary would blame you for attempting to cross the street.

### Rationality vs. Perfection

- Rationality maximizes expected performance, whereas perfection maximizes actual performance. Thus, rationality does not equate to perfection.
- Retreating from the requirement of perfection is not just about fairness to agents. It is essential to design agents that can perform well under given conditions, as expecting an agent to do the best action after the fact is unfeasible unless we improve the accuracy of prediction tools like crystal balls or time machines.

- The example of the cargo door incident on a passing airliner illustrates that rational actions can still be effective despite unforeseeable events.
